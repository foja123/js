syntax parser:

un programma che legge il codice e determina cosa fa e se è stato scritto grammaticamente bene.

lexical environment:

dove qualcosa risiede nel codice scritto. function hello () {var a = "Hello"} In questo caso a risiede dentro la funzione.

execution context:

Il lexical environment che è attualmente in esecuzione

global execution context

crea ogni volta il global object e il this. A livello globale il global object (window) = this. Sono allo stesso livello. Se creo una variabile globale (quindi non dentro una funzione), oppure una funzione esse vengono automaticamente attaccate al global object (window).

execution context
CREATION PHASE: crea il global object, il this, outer environment, e setta lo spazio di memoria per le varibili e le funzioni (HOISTING). Tutto questo prima che il programma venga letto linea per linea. Le funzioni vengono settate in memoria nella loro interezza mentre le varibili no. La variabile viene settata in memoria come undefined finche non viene fisicamente valorizzata.

namespace

Un contenitore per varibili e funzioni. Per mantenere variabili e funzioni con lo stesso nome separate. JS non ha di default in concetto di namespace vero e proprio ma si possono usare dei fake namespace attraverso gli oggetti. Creando oggetti diversi ma con stessi nomi di proprietà riusciamo ad evitare collisioni di variabili e funzioni(metodi) con lo stesso nome.

JSON

JavaScript Object Notation. E' simile ad un oggetto javascript solo che le proprietà vanno messe come stringhe ""

Le funzioni in javascript sono oggetti

FIRST CLASS FUNCTIONS

Ogni operazione che si puo fare con altri tipi di dato si puo fare anche con le funzioni. Assegnarle ad una variabile, passarle e crearle al volo. La funzione è un oggetto con delle proprietà, il codice che noi scriviamo all'interno della funzione è una delle proprietà e può essere invocabile.

Expression.

Codice che restituisce un valore. Non necessariamente deve essere salvato in una variabile.
